## The established parked archive: `../FUTUREPARK/`

The repo already has a large parked-material archive at **`FUTUREPARK/`** (root).
It is the same concept as this folder under an older ALL-CAPS name, and it is kept
**in place on purpose** — it contains embedded git repos (`ECC/`, `aibudgetguard/`
as gitlinks) and `*.docx` data dumps that would be risky to rename. Treat the two
together:

- **`FUTUREPARK/`** — the existing archive (design refs `HoundShield.html`,
  strategy `HERMES_*` / `KAELUS-*`, side projects, duplicate docs). See
  `FUTUREPARK/README.md` for its full manifest.
- **`FutureUse/`** (here) — the canonical name going forward. Park **new**
  future-use material here and add a row to the table above.

> Why two folders? Renaming `FUTUREPARK/` would disturb its embedded gitlinks for
> zero functional gain. So `FutureUse/` is the front door; `FUTUREPARK/` is the
> existing back room. Both are "use someday," neither is junk.

## Rules

1. Future-*use* (might wire in later) → here. Old/superseded → `../OldVersions/`.
   Future *app surfaces* (desktop/mobile/CLI) → `../FutureApp/`.
2. Label every item. An unlabeled park is a lost park.
3. App code never comes here.
