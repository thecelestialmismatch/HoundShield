HoundShield runs as a local or self-hosted reverse proxy. You change one environment variable — your AI tool's API base URL — to point at your HoundShield endpoint. Every outbound prompt is intercepted, scanned, and either forwarded or blocked, all within your network perimeter.

The blocked prompt returns a policy violation message to the user's tool, citing the specific detection rule triggered (e.g., 'ITAR Export Control', 'Contract Number CUI'). The event is logged with a timestamp, the prompt hash (not the content), the matched pattern, and the relevant NIST 800-171 control reference.
