## 1. PDF — the real logo, on every report

**Was:** `drawBadge()` in `lib/reports/pdf-brand.ts` drew a vector "HS" shield (comment literally said
"no embedded image"). `addImage` appeared nowhere; `public/logo.png` was unused.

**Now:** the real doberman-shield mark is compiled in as a base64 PNG (`lib/reports/logo-data.ts`,
generated by `scripts/gen-logo-data.py` — `public/logo.png` flattened onto white, 256×256, ~35 KB) and
embedded via `doc.addImage`, seated in a white chip so it reads on the dark cover band. `drawBadge`
feature-detects `addImage` and keeps the vector shield as a fallback (so the unit-test jsPDF mock still
works). One change covers **every** report: sample, compliance/audit, SSP/POA&M/Evidence, and the /demo
snapshot. Regenerate the mark with `python3 scripts/gen-logo-data.py`.

## 3. Password reset — the loop-breaker: make the invisible failure visible

The reset **code chain is correct and unit-tested**. It kept "not working" because the send route is
enumeration-safe (always returns 200; the UI always says "check your email") so every *operational*
failure was invisible. New: `/api/health` now reports value-free password-reset readiness
(`lib/auth/reset-diagnostics.ts`) — never any secret value, only shape/host:

| health field | meaning |
|---|---|
| `reset_service_role` | `SUPABASE_SERVICE_ROLE_KEY` present + plausible (needed by `admin.generateLink`) |
| `reset_resend` | `RESEND_API_KEY` present + `re_…` shape (the actual sender) |
| `reset_app_url` | `NEXT_PUBLIC_APP_URL` sanity — flags `kaelus.ai`, localhost, `*.vercel.app` |
| `reset_app_url_hint` | value-free next step when app_url is default/suspect |
| `reset_sender_domain` | the FROM domain that must be a verified Resend sender |

Forgot-password success screen now adds an honest "Didn't get it? Check spam, or contact support" line.
