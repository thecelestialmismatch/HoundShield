## 1. Revenue Blockers (Critical — Fix Today)

### B1: Stripe Webhook URL is Wrong
- **Current URL:** Likely still `kaelus.online/api/stripe/webhook`
- **Required URL:** `https://houndshield.com/api/stripe/webhook`
- **Impact:** Subscriptions don't activate. Paid users see free tier. No revenue lands.
- **Fix:** dashboard.stripe.com/webhooks → update endpoint URL

### B2: STRIPE_WEBHOOK_SECRET Missing
- **Current state:** Not set in Vercel
- **Impact:** Even with correct URL, webhook validation fails → `400 Forbidden` → subscription never activates
- **Fix:** After updating webhook URL, copy new signing secret → Vercel env var `STRIPE_WEBHOOK_SECRET`

### B3: OPENROUTER_API_KEY Missing
- **Current state:** Not set in Vercel
- **Impact:** Brain AI returns "trouble connecting" on production. `/api/brain/query` fails.
- **Fix:** Create API key at openrouter.ai → set `OPENROUTER_API_KEY` in Vercel

### B4: Supabase Migrations 003 + 004 Not Pushed to Production
- **Current state:** Only migrations 001+002 are in production DB
- **Impact:** Features added in migrations 003+004 will fail silently on prod (missing columns, tables, or views)
- **Fix:** `cd compliance-firewall-agent && npx supabase db push` with prod env vars
